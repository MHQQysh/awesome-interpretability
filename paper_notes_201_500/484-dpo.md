# 484. Direct Preference Optimization: Your Language Model is Secretly a Reward Model

- **Authors:** Rafael Rafailov, Archit Sharma, Eric Mitchell, Stefano Ermon, Christopher Manning, Chelsea Finn
- **Venue / year:** NeurIPS 2023
- **Paper:** [arXiv:2305.18290](https://arxiv.org/abs/2305.18290)
- **Tags:** `alignment` `preference-learning` `reward-model` `interpretability`

## Introduction
RLHF 通常需要单独训练 reward model、再用 PPO 优化 policy，流程复杂、超参敏感且 reward 不易解释。DPO 问能否把 preference objective 直接写成 language-model classification loss，同时保留 RLHF 的最优 policy。

## Method / Framework
作者从 KL-constrained reward maximization 推导 optimal policy 与 reference model 的关系，把隐式 reward 写成 policy/reference log-prob ratio，再将 Bradley-Terry preference loss 直接优化在 chosen/rejected response 上。DPO 不显式训练 reward model、不在线采样 PPO trajectory，却可解释为让模型提高相对偏好答案的 log likelihood。

## Baselines / Comparisons
比较 PPO-based RLHF、显式 reward-model 方法、SFT、IPO/KTO 等后续 preference objectives，覆盖 sentiment control、summarization preference 和 dialogue alignment。核心 metric 是 preference accuracy、human preference、generation quality 与训练稳定性。

## Experiments / Findings
DPO 在论文的 sentiment、TL;DR summarization 和 dialogue 实验中达到或超过 PPO/RLHF，同时实现更简单、训练更稳定。分析显示 policy 的 log-ratio 本身携带 reward signal，因此“language model secretly a reward model”不是额外解释器，而是 preference loss 的数学重参数化。

## Ablation / Error Analysis
reference model、beta/KL 权重、训练 epochs 和数据质量决定 preference/quality trade-off；过度优化会出现长度偏置、mode collapse 或偏好分布外推失败。DPO 的 offline nature 避免 PPO 的 rollout instability，但也继承偏好数据噪声和 reference bias。

## Limitations
理论依赖 Bradley-Terry、KL 正则和 preference 数据假设；实际人类偏好可能非 transitive、多目标且上下文依赖。DPO 的可解释 reward 是训练目标的等价量，不等于模型生成时真正使用了人类可读的理由。

## 两句话总结
DPO 将 RLHF 的 KL-regularized preference optimization 化成直接比较 chosen/rejected log-prob 的 loss，省去显式 reward model 与 PPO。它提高了对齐训练的稳定性和可分析性，但 preference data、reference bias 和分布外行为仍是主要风险。
