# 495. Large Language Models: Testing Their Capabilities to Understand and Explain Spatial Concepts (Short Paper)

- **Authors:** Majid Hojati, Rob Feick
- **Venue / year:** COSIT 2024, LIPIcs
- **Paper:** [DOI 10.4230/LIPIcs.COSIT.2024.31](https://doi.org/10.4230/LIPIcs.COSIT.2024.31)
- **Tags:** `evaluation` `spatial-reasoning` `SQL-generation` `explanation`

## Introduction
The paper asks whether fluent LLM responses about geography reflect usable spatial reasoning. It targets a practical gap: a model may explain a GIS concept plausibly while failing the calculation, or generate a SQL query that looks reasonable but uses a nonexistent function. The authors therefore design an extensible, reproducible test suite that evaluates both how-to explanations and executable spatial SQL.

## Method / Framework
The study creates 96 questions covering area, angle, bounding box, distance, proximity, projection, geometry/format, graph/network, center, matrix translation, error/accuracy, validation, interpolation/variogram, 3D coordinates, geohash, direction, spatial topology, midpoint, and dimension. Each question is written in two forms: a methodological question asking how to solve the task, and a PostGIS question asking for SQL. Calls are cold-started through LangChain so previous model conversations cannot contaminate later answers.

Four models are compared: GPT-3.5, GPT-4, Llama-2-7B, and Falcon-40B. Six tests score whether an answer is produced, whether a final answer is correct, whether the how-to methodology is correct, whether SQL can be parsed, whether the query runs in PostGIS, and how much editing is needed. Scores are 1 for correct and .5 for almost correct, with numeric reference answers generated by the authors' database queries.

## Baselines / Comparisons
This is a cross-model evaluation rather than a trained-method comparison. GPT-3.5 and GPT-4 represent closed enterprise models; Llama-2-7B and Falcon-40B are open models. The two task formats are also controlled baselines: natural-language how-to explanations versus structured SQL generation. The authors use human inspection for methodology and database execution for SQL, which is more informative than judging all responses by surface fluency.

## Experiments / Findings
GPT-4 is best overall, GPT-3.5 is second, and the open models trail substantially. Across the question set, all models answer at least 90% of prompts, but correct final answers are about 75% for GPT-4, 61% for GPT-3.5, 18% for Falcon, and 3% for Llama-2. Correct how-to methodology is about 89%, 74%, 41%, and 20%, respectively. For SQL that runs without edits, GPT-4 reaches 95%, GPT-3.5 89%, Falcon 24%, and Llama 6%.

Structured SQL is easier than vague methodology: the paper reports an aggregate advantage of about 46% for valid spatial SQL generation. All models handle basic geometry distinctions, direction, angles, topological relations, latitude/longitude, and WKT/GeoJSON formats better than they compute the final correct value. GPT-4 and GPT-3.5 are close on SQL questions even when their overall spatial reasoning differs.

## Ablation / Error Analysis
The two prompt forms are the core ablation, and the six-test breakdown separates answer production, semantic correctness, parsing, execution, and editability. Hallucinated functions are a concrete failure mode: Falcon uses nonexistent names such as PostGIS_Distance or rhumb_line_distance in about 13% of cases, while both GPT models are around 1%. Falcon can also answer a place-name question when asked to compute a centroid, showing confusion between the task wording and the real-world topic.

## Limitations
The question set is only 96 items, covers selected GIS concepts, and many answers depend on explicit coordinate/data assumptions. Manual scoring of how-to explanations remains subjective, while SQL execution only measures the constructed database problem, not real GIS usefulness. The short paper does not test fine-tuning, longer conversations, multilingual geography, map images, or robustness to prompt paraphrase.

## Two-sentence summary
LLMs are much more reliable when spatial questions are converted into constrained PostGIS SQL than when asked for an open-ended methodological explanation. GPT-4 is strongest, but even it can be wrong on advanced concepts and arithmetic, so geospatial use requires executable checks rather than trusting fluent explanations.
