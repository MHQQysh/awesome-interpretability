# 481. TPU v4: An Optically Reconfigurable Supercomputer for Machine Learning with Hardware Support for Embeddings

- **Authors:** Norman P. Jouppi et al.
- **Venue / year:** ISCA 2023
- **Paper:** [arXiv:2304.01433](https://arxiv.org/abs/2304.01433)
- **Tags:** `LLM-systems` `hardware` `embeddings` `scaling`

## Introduction
LLM 的模型规模、Transformer 通信和 embedding workload 把 TPU supercomputer 从 256 节点推到 4096 节点，同时带来主机失效、all-to-all 带宽和 checkpoint/restart 问题。论文解释 TPU v4 如何在硬件层支撑这类模型，并提醒 interpretability 研究也受训练基础设施和可复现实验条件影响。

## Method / Framework
系统由可重构 Optical Circuit Switches (OCS)、3D torus/topology search 和 SparseCore embedding accelerator 构成。OCS 通过毫秒级 MEMS mirror 改变光链路、绕过不可用 CPU host；SparseCore 处理稀疏 embedding lookup/combination；ML 搜索联合优化 DNN parallelism、slice topology 和 hardware。

## Baselines / Comparisons
比较 TPU v4/v3、Graphcore IPU Bow、Nvidia A100，以及 OCS/static topology、不同 slice geometry 和 DLRM/Transformer production workload。论文还用 TPUv4 的 4K-chip supercomputer 与不同 availability/scale 组合比较 goodput。

## Experiments / Findings
TPUv4 相对 TPUv3 速度 2.1x、performance/W 2.7x；4096 chips 约近 10x overall throughput。SparseCore 对 embedding model 加速 5--7x，只占约 5% die/power；OCS 让系统容忍 0.1--1.0% unavailable host。对相同规模系统，TPUv4 约比 Graphcore 4.3--4.5x 快，比 A100 1.2--1.7x 快且功耗低。

## Ablation / Error Analysis
固定与可重构 topology、regular/twisted torus、不同 data/model/pipeline parallel mapping 是主要系统 ablation；topology 与模型通信维度匹配可带来约 1.2--2.3x 差异。失败 host、slice size 和 all-to-all embedding communication 是影响 goodput 的关键因素。

## Limitations
这是 Google production system 论文，硬件/软件细节和 workload 选择带有平台特定性；性能比较不等价于所有 LLM 训练。它不是解释算法，但说明大规模 interpretability experiment 的计算、通信和能耗基础。

## 两句话总结
TPUv4 通过 OCS 可重构网络和 SparseCore 把大规模 LLM/embedding 的可靠性、通信和效率问题一起处理。其贡献是系统可扩展性而非模型解释，不过它直接决定大模型内部分析能否重复运行和扩大规模。
